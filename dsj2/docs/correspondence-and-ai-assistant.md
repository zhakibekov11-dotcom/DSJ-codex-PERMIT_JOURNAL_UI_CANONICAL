# Письма, КП и AI-помощник

## Что добавлено

- административный раздел `Письма и КП`
- реестр исходящей корреспонденции по компании
- письмо и коммерческое предложение как отдельные типы документов
- несколько получателей в одном документе
- фиксация дат отправки по каждому получателю
- Word-экспорт на фирменном бланке `Stroy Company 2030`
- AI-помощник для:
  - генерации черновика
  - улучшения текста
  - анализа письма перед отправкой

## Модели

- `Correspondence`
- `CorrespondenceRecipient`

## Как работает отправка сейчас

- письмо создаётся в реестре со статусом `DRAFT`
- по кнопке `Отправить всем` система проходит по списку получателей
- если у получателя указан email, отправка помечается как `SENT`
- если email не указан, получатель получает статус `FAILED`
- общий документ получает статус:
  - `READY_TO_SEND`, если не отправлено ни одному
  - `PARTIALLY_SENT`, если часть получателей отправлена
  - `SENT`, если отправлены все

Сейчас это mock-ready поток без внешнего SMTP/ESP. Архитектура подготовлена для подключения реальной доставки позже.

## AI-помощник

Если `OPENAI_API_KEY` задан:

- backend обращается к OpenAI Responses API
- модель берётся из `OPENAI_MODEL` или по умолчанию `gpt-5-mini`

Если `OPENAI_API_KEY` не задан:

- включается безопасный fallback
- система всё равно помогает шаблонным черновиком и базовым анализом

## Что логируется

В аудит попадают:

- `correspondence.created`
- `correspondence.sent`

## Что можно развить на следующем этапе

- редактирование уже сохранённого письма
- реальная отправка через SMTP / Resend / Microsoft 365
- отдельные шаблоны писем и КП по типам
- сохранение AI-версий и комментариев как истории редакций
