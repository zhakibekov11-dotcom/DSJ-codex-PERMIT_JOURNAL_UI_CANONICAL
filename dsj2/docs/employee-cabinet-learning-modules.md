# Расширение кабинета сотрудника: документы, удостоверения, обучение, тестирование

## Что добавлено

- `EmployeeDocument` для документов сотрудника и подтверждений прохождения.
- `SafetyCertificate` для удостоверений по ТБ.
- `TrainingProgram` и `TrainingAssignment` для назначения обучения сотруднику.
- `Exam`, `ExamQuestion`, `ExamOption`, `ExamAttempt` для MVP-тестирования.

## Основная логика

- Инструктажи остаются основным сценарием продукта и не зависят от обучения.
- Обучение и тестирование встроены в тот же личный кабинет сотрудника как дополнительный блок.
- Сотрудник видит только свои документы, удостоверения, обучения и тесты.
- Администратор и инженер по ТБ управляют новыми сущностями через отдельные простые разделы без редизайна продукта.

## Связи между сущностями

- `TrainingProgram` описывает программу обучения и материалы.
- `TrainingAssignment` связывает программу с сотрудником и хранит прогресс.
- `Exam` привязывается к `TrainingProgram` в формате 1:1.
- `ExamAttempt` привязывается к `TrainingAssignment` и хранит результат попытки.
- После успешного завершения обучения могут автоматически создаваться:
  - `EmployeeDocument`
  - `SafetyCertificate`

## Принцип генерации артефактов

- Если `TrainingProgram.createsDocument = true`, после завершения создаётся документ `COMPLETION_CONFIRMATION`.
- Если `TrainingProgram.createsSafetyCertificate = true`, после завершения создаётся:
  - документ типа `SAFETY_CERTIFICATE`
  - запись `SafetyCertificate`

Это оставляет совместимую модель для будущей реальной генерации PDF, шаблонов и интеграции с внешними реестрами.
